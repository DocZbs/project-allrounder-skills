# Project Deck Router

Choose one primary scene. Add material from another scene only when it directly serves the primary decision.

## Proposal Defense

**Decision:** fund/select the project.

**Arc:** need and stakes → precise problem → objectives → novelty/value → work packages and route → evidence of feasibility → team/resources → indicators → schedule/risk/budget → requested decision.

Do not present expected results as completed evidence.

## Kickoff or Opening Review

**Decision:** approve the executable baseline and collaboration model.

**Arc:** authority and scope → objectives/indicators → deliverables → WBS and interfaces → roles/RACI → milestones → quality/evidence plan → risk/change/communications → immediate actions.

## Progress or Midterm Review

**Decision:** accept status and approve corrective/next-stage actions.

**Arc:** original baseline → evidence-backed completion → representative results → indicator trend → schedule/budget variance → issues/risks/changes → recovery or next-stage plan → decisions needed.

Separate `completed with evidence`, `in progress`, `delayed`, and `changed by approval`.

## Technical Review or Demonstration

**Decision:** accept the technical approach, maturity, or next release.

**Arc:** operating problem → requirements → architecture/method → critical implementation → test design → results and comparison → demonstration → limitations/risk → release or remediation decision.

## Acceptance Defense

**Decision:** accept deliverables/indicators or define remediation.

**Arc:** acceptance authority → scope and criteria → deliverables inventory → indicator-by-indicator target/method/result/evidence/judgment → application or benefit evidence → finance/handover status where required → unresolved items → explicit acceptance request.

Never infer `passed` without the authorized criterion and evidence.

## Closeout Presentation

**Decision:** confirm closure and preserve reusable learning.

**Arc:** original mandate → final scope and changes → objective/indicator completion → representative deliverables and impact → contractual/program and finance closure → handover/archive → lessons → follow-on opportunities → closure decision.
