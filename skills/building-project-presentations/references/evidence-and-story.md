# Evidence, Story, and Reference Decks

## Reference-Deck Protocol

Use at least one authorized/open or user-authorized same-scene deck when available. Record:

`source | owner/publisher | authorization/provenance | year | scene | audience | decision | narrative | evidence density | visual rhythm | transferable principle | non-transferable content`

Learn from sequencing, evidence selection, transitions, time allocation, and likely questions. Do not copy protected wording, figures, data, layouts, icons, photos, or branding. Old decks never override current requirements.

If no legitimate same-scene deck is available, record the gap and use current official review criteria plus the installed presentation skill's narrative/design rules.

## Claim-to-Evidence Contract

For every substantive slide, define:

- claim the audience should retain;
- controlling requirement or decision question;
- evidence/source and version;
- evidence limitations and uncertainty;
- visual form that makes the comparison visible;
- implication for the current decision;
- speaker-note caveat and anticipated question.

## Indicator Slide Contract

An indicator judgment needs:

`criterion source → target → definition → measurement method → sample/period/environment → exclusions → observed result → evidence path → authorized judgment`

If any required element is missing, label the result `UNVERIFIED` or `EVIDENCE GAP`; do not show a green pass marker.

## Evidence Hierarchy

Prefer original measured data, signed/official acceptance records, test reports, approved baselines, versioned deliverables, and traceable source artifacts. Use summaries only when they link back to the underlying evidence.

## Natural Visible Copy

- Write for the audience, not the production team.
- Prefer concrete claims and active verbs.
- Explain why the evidence changes the decision.
- Put methodological detail in notes or appendix unless it is central to trust.
- Keep caveats visible when they materially change interpretation.
