# Lifecycle and Exit Gates

Use the smallest stage that matches the current decision. Do not skip an exit gate because a deadline is close.

| Stage | Required work | Minimum outputs | Exit gate |
|---|---|---|---|
| Intake | Inventory sources, versions, owners, confidentiality, and missing inputs | Source register, gap list | Controlling sources confirmed or gaps explicitly accepted |
| Opportunity | Test eligibility, fit, value, capacity, schedule, budget, and risk | Go/no-go note | Authorized owner records a decision |
| Requirements | Decompose call, scorecard, task book, or contract | Requirement-response matrix | Every requirement has response, owner, and status |
| Proposal | Build original argument/solution and evidence | Proposal package and audit | Facts, form, budget, attachments, and approvals pass |
| Award handoff | Convert approved material into execution baseline | Baseline, indicator register, WBS | Scope, indicators, deliverables, budget, and dates reconciled |
| Planning | Define work packages, RACI, milestones, resources, quality, and risk | Executable plan | Owners accept tasks and gates |
| Execution | Perform research, development, testing, fieldwork, or consulting | Versioned outputs and raw records | Outputs meet task-level definition of done |
| Control | Review schedule, scope, quality, finance, issues, risks, and changes | Status and registers | Material variance has an action or approved change |
| Midterm | Compare baseline with verified progress | Report, evidence pack, deck | Every completion claim links to evidence |
| Acceptance prep | Test each indicator and deliverable | Acceptance matrix and remediation list | No unsupported acceptance claim remains |
| Acceptance | Present, demonstrate, answer, record findings, and remediate | Signed/official evidence and action log | Findings are closed or formally carried |
| Handover | Transfer artifacts, access, data, IP constraints, operations, and training | Handover inventory and sign-off | Recipient confirms completeness and access |
| Closeout | Reconcile technical, contractual/program, finance, archive, and learning | Closeout pack and archive index | All required closures are evidenced and authorized |

## Recovery Route

For a project already in progress, do not pretend the planned baseline is the actual state:

1. Freeze the controlling baseline.
2. Reconstruct actual artifacts, expenses, decisions, and dates.
3. Compare baseline with actual state.
4. Classify each difference as correction, variance, risk, issue, or approved/unapproved change.
5. Build a recovery plan with owners and decision deadlines.
