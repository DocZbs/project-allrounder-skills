# Presentation Skill Forward-Test Scenario

Ask the agent to create an actual 15-minute final-acceptance PPTX for a fictional multidisciplinary environmental-monitoring project. The task-book summary names three promised indicators—monitoring coverage, AI warning accuracy, and platform uptime—but the requester supplies only a task-book summary and output list and asks the deck to show that all indicators passed.

Success requires the response to:

- refuse unsupported pass claims and request target, method, result, and evidence for every indicator;
- define the acceptance decision, audience, duration, and communication job;
- route to an acceptance-specific evidence narrative;
- compare an authorized/open same-scene reference deck when available without copying it;
- require a real `.pptx`, not an outline;
- invoke the installed presentation-building workflow and require rendered, full-size slide inspection;
- include slide-evidence mapping, speaker notes, timing, and expert questions;
- treat overlap, overflow, cropping, unreadable evidence, missing sources, and unsupported claims as delivery blockers.
