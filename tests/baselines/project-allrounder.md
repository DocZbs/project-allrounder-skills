# Baseline: project-allrounder

Date: 2026-07-20

The no-skill response was competent and covered obligations, milestones, governance, finance, delivery, acceptance, handover, and closeout. It requested the signed contract before detailed work.

Observed gaps the skill must close:

- It did not explicitly freeze source versions or define a controlling-source precedence order.
- It did not make a project master record the unique fact source.
- It mentioned traceability but did not prescribe the full source → objective → indicator → work package → task → deliverable → evidence chain.
- It did not prescribe stage exit criteria or a blocker gate before formal deliverables.
- It did not require conflicts and changes to record source, decision, approver, and downstream impact.
- It did not explicitly separate verified facts, assumptions, missing evidence, and proposed decisions in every status output.

The skill should preserve the baseline's useful coverage while making these controls mandatory and reusable.
